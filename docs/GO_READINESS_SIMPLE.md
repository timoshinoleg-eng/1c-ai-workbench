# GO readiness — простыми словами

## Решение

**GO для поиска первого клиента и проведения демонстрации.**

Не обещаем “волшебный AI, который сам всё исправит”. Обещаем понятную вещь:

> Быстро найти место в 1С, показать файл, показать фрагмент, объяснить как проверить руками.

## Что реально готово

- Есть рабочий локальный стенд.
- Есть демо-пример 1С.
- По демо-примеру построен быстрый поиск.
- Есть готовая страница для демонстрации: `demo-showcase\index.html`.
- Есть 3 простых demo-вопроса.
- Есть готовые ответы с доказательствами.
- opencode видит локальный инструмент `1c-code-index` и показывает `connected`.
- Есть security checklist.
- Есть коммерческий оффер и цена.

## Что показывать клиенту

1. “Вот пример 1С”.
2. “Вот вопрос: где считается сумма?”
3. “Вот найденный файл”.
4. “Вот конкретное место”.
5. “Вот как это проверить руками”.
6. “Мы ничего не меняли в базе”.

## Что не обещать

- Не обещать, что AI сам исправит 1С.
- Не обещать полную замену 1С-разработчика.
- Не обещать 152-ФЗ “из коробки”.
- Не использовать свой облачный AI-ключ для данных клиента.
- Не просить клиента прислать выгрузку себе в Telegram/email/cloud.

## Условия безопасного платного пилота

Перед работой с клиентской 1С:

1. Клиент делает копию/выгрузку у себя.
2. Всё запускается на компьютере клиента.
3. Если используется облачный AI, ключ вводит клиент.
4. Результат AI проверяет человек.
5. Исходная рабочая база 1С не меняется.

## Команды для демо

Открыть демо-страницу:

```powershell
cd C:\1c-ai-workbench
.\scripts\11_open_demo_showcase.ps1
```

Проверить, что стенд живой:

```powershell
cd C:\1c-ai-workbench
.\scripts\06_healthcheck.ps1
```

Показать настоящий поиск:

```powershell
cd C:\1c-ai-workbench
.\tools\code-index-mcp\target\release\bsl-indexer.exe grep-body --path "C:\1c-ai-workbench\generated\index\source-mirror" --pattern "Движения.Продажи" --language bsl --limit 20
```

Проверить opencode:

```powershell
cd C:\1c-ai-workbench
opencode mcp list
```

Должно быть: `1c-code-index connected`.
