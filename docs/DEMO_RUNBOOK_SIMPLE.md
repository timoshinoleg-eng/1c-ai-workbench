# Demo runbook — простыми словами

## Что показываем

Мы показываем не саму 1С как бухгалтерскую программу, а помощника, который быстро находит место в настройках и коде 1С.

Демо отвечает на вопросы вида:

- где считается сумма документа;
- где документ пишет продажи;
- где находится нужная форма;
- какой файл открыть, чтобы проверить ответ руками.

## Что уже готово

- Демо-пример 1С уже лежит в `C:\1c-ai-client\dump`.
- Поиск по этому примеру уже построен.
- Проверка стенда проходит зелёным.
- opencode может подключить локальный поиск через MCP.

## Как запустить проверку стенда

```powershell
cd C:\1c-ai-workbench
.\scripts\06_healthcheck.ps1
```

Если всё хорошо, будут строки `[OK]`.

## Как открыть opencode с этим проектом

```powershell
cd C:\1c-ai-workbench
opencode .
```

В вопросе явно проси использовать инструмент:

```text
Используй 1c-code-index. Найди в демо 1С, где документ ЗаказНаряд считает сумму. Ответь простыми словами: что найдено, какой файл открыть, какой фрагмент проверить руками.
```

## Что показать клиенту за 5 минут

1. Открыть папку `C:\1c-ai-workbench`.
2. Запустить `06_healthcheck.ps1` и показать `[OK]`.
3. Открыть opencode.
4. Задать вопрос про `ЗаказНаряд`.
5. Показать, что ответ содержит путь к файлу и понятное объяснение.
6. Открыть файл из ответа или показать готовый эталон `demo-answers\golden_answers_summary.md`.

## Безопасность простыми словами

- Мы не подключаемся к рабочей базе клиента.
- Работаем с копией/выгрузкой.
- Поиск строится локально на компьютере клиента.
- Файлы 1С не отправляются исполнителю.
- Если используется облачная AI-модель, ключ должен быть ключом клиента.
- AI не меняет 1С, только показывает где искать.
