# Стандартные демо-ответы 1C AI Dev Workbench

Локальный AI-навигатор по вашей 1С-конфигурации находит объект, файл и место для ручной проверки.

## 1. Где считается сумма заказ-наряда?

**Короткий вывод:** сумма заказ-наряда считается на форме документа, когда пользователь меняет услугу или количество.

**Что найдено:**
- Документ: `ЗаказНаряд`
- Файл: `Documents/ЗаказНаряд/Forms/ФормаДокумента/Ext/Form/Module.bsl`
- Места: `обновитьСуммуПоТекущейСтрокеУслуг`, `обновитьСуммуДокумента`

**Простыми словами:** одна часть считает сумму строки как цена × количество, другая складывает все строки в итог документа.

**Как проверить руками:** открыть файл, найти `Цена`, `Количество`, `СуммаДокумента`; в форме документа поменять количество услуги и посмотреть, меняется ли сумма.

**Уверенность:** высокая — найдено точное место расчёта.

**Что не найдено:** отдельный расчёт суммы при печати не нужен: печать берёт уже готовую сумму.

## 2. Где записываются продажи?

**Короткий вывод:** продажи записывает документ `Реализация`.

**Что найдено:**
- Документ: `Реализация`
- Файл: `Documents/Реализация/Ext/ObjectModule.bsl`
- Место: строки с `Движения.Продажи`

**Простыми словами:** когда документ проводят, он добавляет данные в “журнал продаж”: что продали, кому, сколько и на какую сумму.

**Как проверить руками:** открыть файл, найти `Движения.Продажи`, провести документ в демо-базе и посмотреть движения документа.

**Уверенность:** высокая — найдено точное место записи продаж.

**Что не найдено:** проблема с конкретными цифрами требует проверки конкретного документа и данных клиента.

## 3. Как получить контрагентов на букву “А”?

**Короткий вывод:** отдельного готового отчёта в демо-базе нет, но есть список контрагентов, где можно сделать отбор.

**Что найдено:**
- Объект: `Контрагенты`
- Файл списка: `Catalogs/Контрагенты/Forms/ФормаСписка/Ext/Form.xml`
- Поля: наименование, код, тип, телефон, договор, комментарий

**Простыми словами:** для разовой задачи можно открыть список клиентов/поставщиков и поставить фильтр “название начинается на А”.

**Варианты улучшения:**
1. Дешёвый: инструкция пользователю, как поставить фильтр.
2. Удобный: отдельный отчёт с выбором буквы.
3. Бизнесовый: отчёт для обзвона с фильтрами “есть телефон”, “тип клиента”, “есть договор”.

**Как проверить руками:** открыть список `Контрагенты`, включить фильтр по названию, проверить поля в списке.

**Уверенность:** средняя — список найден точно, но отдельный отчёт надо делать как доработку.

**Что не найдено:** готовый отдельный отчёт “Контрагенты на букву А”.

## 4. Что затронет новое поле в товаре/услуге?

**Короткий вывод:** начинать надо с карточки товара/услуги, потом проверить документы и отчёты.

**Что найдено:**
- Объект: `Номенклатура`
- Файлы: `Catalogs/Номенклатура.xml`, формы элемента и списка

**Простыми словами:** если добавить новое поле, его нужно не только создать, но и показать пользователю, а потом проверить места, где товары используются.

**Как проверить руками:** открыть карточку товара, список товаров, документы продажи/поступления/заказ-наряд.

**Уверенность:** средняя — найден план проверки, но бизнес-правила надо уточнить у владельца процесса.

**Что не найдено:** без требований бизнеса нельзя точно сказать, где поле должно участвовать.
