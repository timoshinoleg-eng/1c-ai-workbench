# /trace-flow — Трассировка потока выполнения

**Когда использовать:** нужно понять, какие функции вызываются от точки входа до результата — построить цепочку вызовов.

**Описание:** рекурсивно строит цепочку вызвов от указанной функции: кто вызывает и что она вызывает.

**Шаги:**
1. Определить точку входа (функция/процедура)
2. Получить callees через `get-callees`
3. Для каждого callee — рекурсивно получить его callees (до глубины 3-4)
4. Построить дерево вызовов
5. Отметить ветки с потенциальными проблемами (длинные цепочки, рекурсия)

**Формат ответа:**
```
Точка входа: <функция> в <файл>

Дерево вызовов (глубина до 3):
<функция>
├── <callee 1> (<файл>)
│   ├── <callee 1.1> (<файл>)
│   └── <callee 1.2> (<файл>)
├── <callee 2> (<файл>)
│   └── <callee 2.1> (<файл>)
└── <callee 3> (<файл>)

Замечания:
- Глубина цепочки: N
- Потенциальная рекурсия: да/нет
- Длинные ветки (>5): <список>
```

**Уверенность:** высокая для точных имён, средняя при неоднозначности.
