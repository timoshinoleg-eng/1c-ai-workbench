# /explain-module — Разбор модуля 1С

**Когда использовать:** нужно быстро понять, что делает модуль, без чтения всего кода.

**Описание:** режим строит evidence-first summary по модулю 1С: назначение, экспорты, ответственность, риски.

**Формат ответа:** expected_answer_format (короткий вывод → объекты → файл → процедура → уверенность → проверка)

**Шаги:**
1. Найти модуль через `get-file-summary` или `search-function`
2. Извлечь первые 200 строк для анализа заголовка и экспортов
3. Собрать список экспортируемых процедур/функций
4. Определить предполагаемое назначение по неймингу и обращениям к объектам метаданных
5. Отметить риски: длинные процедуры, отсутствие комментариев, сложные вложенности
6. Собрать ответ по evidence-first формату

**Пример:**
```
Короткий вывод: модуль отвечает за расчёт суммы заказ-наряда при вводе услуг.
Объект: Документ.ЗаказНаряд
Файл: Documents/ЗаказНаряд/Forms/ФормаДокумента/Ext/Form/Module.bsl
Процедуры: обновитьСуммуПоТекущейСтрокеУслуг, обновитьСуммуДокумента
Уверенность: высокая
```
